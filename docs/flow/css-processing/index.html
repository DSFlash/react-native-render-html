CSS Processing

There are significant differences between the CSS standard and how styles are handled in React Native. Most notably, â€‹Text styles don't inherit from â€‹View styles. The reconciliation is handled by the â€‹TRE.

Inheritance#

As stated before, a React Native â€‹View cannot receive textAlign style. It could even crash the native app. The â€‹@native-html/css-processor and â€‹@native-html/transient-render-engine libraries handle inheritable CSS properties by merging inheritable properties in the â€‹TRT.

Translation#

Every CSS property has a validator registered in the CSS processor, in charge of validating values for those rules. Validators are multipurposed:

  1. Handle special units such as absolute (pt), relative (rem) and keywords such as small (font-size) and thin (for borders) and translate those values in DIP (device independent pixels).

  2. Group properties by inheritability, native target (block for â€‹View and text for â€‹Text), and compatibility (native for properties translatable as native styles and web for other properties).

  3. Discard properties which values are invalid.

important

Note that special inherit, initial and unset values are not supported.

Mixed Styles Declaration#

Mixed styles declarations (â€‹MixedStyleDeclaration) are a blend between React Native styles (ViewStyle, TextStyles) and CSS properties such as those handled by the CSS processor. This format enables features that cannot be handled directly by React Native style engine. For example â€‹white-space collapsing, â€‹list-style-type, font selection (see â€‹Textual page) or â€‹object-fit support in images (see â€‹Images page).

A mixed styles declaration.
const mixedStyles = {
whiteSpace: 'normal',
color: 'gray'
}

Specificity#

Despite its lack of complex selectors, CSS specificity will be honoured by this library. Most notably, a CSS property will be resolved by applying the following precedence in ascending order (latest will override formers):

  1. Inherited styles

  2. User Agent styles (see â€‹Transient Render Engine, element model)

  3. Tags styles (â€‹tagsStyles prop)

  4. Classes styles (â€‹classesStyles prop)

  5. IDs styles (â€‹idsStyles prop)

  6. Inline styles (â€‹style DOM node attribute)

So inline styles will take precedence over IDs styles, which will take precedence over classes styles, ...etc.
