Custom Rendering

The renderer API shipped since Foundry (v6) is at the same time more strict and more flexible. To get ready for this new API, you must understand some basics of the transient render tree produced by the â€‹TRE:

  1. During the transient render tree generation, every DOM node is translated to a â€‹TNode.

  2. â€‹TText nodes correspond to DOM â€‹Text nodes (anonymous â€‹TText nodes) or DOM elements which children are DOM â€‹Text nodes (named â€‹TText nodes). So a â€‹TText node cannot have children, and its content is a string accessible with the â€‹data field.

  3. Thanks to hoisting, â€‹TPhrasing nodes can only have â€‹TText and â€‹TPhrasing nodes as children.

  4. â€‹TBlock nodes can have any children.

tip

You are kindly advised to read â€‹Transient Render Engine page before continuing!

Options for Custom Rendering#

You can customize rendering at two steps of the flow:

  1. During â€‹TRT generation. via HTML model definition.

  2. At (React) render time via custom renderers.

These customizations are not exclusive, so you can use both approaches at the same time.

Renderer Props Summary#

A custom renderer will receive the below props:

â€‹tnode

The â€‹TNode to render.

â€‹TDefaultRenderer

The default fallback renderer for this â€‹TNode.

â€‹InternalRenderer

The internal renderer for this tagName. An internal renderer is like a custom renderer, but registered internally. If there is no internal renderer registered for this tag, â€‹InternalRenderer will be equal to â€‹TDefaultRenderer.

â€‹style

The flatten style object which should be passed to the root element returned by this component.

â€‹textProps

To use when you render a â€‹Text-based element (e.g. the â€‹type prop is "text").

â€‹viewProps

To use when you render a â€‹Text-based element (e.g. the â€‹type prop is "block").

â€‹type

To check whether a â€‹Text ("block") or â€‹View ("text") is expected as the root element returned by this component.

â€‹propsFromParent

Props passed directly from the parent custom renderer via â€‹TChildrenRenderer. See â€‹propsForChildren prop.
