Images

note

This article covers the <img> element renderer. <picture> is not yet supported.

Scaling#

Inline Styles#

The renderer will automatically scale images down to the available width, even when the provided inline style width is greater than the container width.

important

You are strongly advised to provide a contentWidth property from useWindowDimensions official hook to help this component handle the scaling.

Object Fit#

This library supports object-fit CSS property. In the below example, which is very similar to the previous one, we set explicitly object-fit:Â contain; to letterbox the image in the container of width 50% and height 100px.

tip

object-fit is a mixed style and thus you can use it in mixed styles declarations such as tagsStyles.

Automatic Sizing#

The next image will be sized automatically thanks to the contentWidth and computeEmbeddedMaxWidth props. The latter allows you to set the maximum width from contentWidth, or disabling scaling by returning Infinity.

Preloading#

Similarly to browsers, this library will place a print box before fetching image dimensions when both width and height attributes are provided, or the two dimensions are set in the style attribute. This is great to avoid images "jumping" from zero height to their computed height, and is a hint to good web design.

Via useInternalRenderer#

useInternalRenderer has a great advantage over using InternalRenderer prop: you have access to the internal component props. In this scenario, we are going to display an interactive thumbnail which will show the full resolution image when pressed. To do so, we are going to read the URI from the internal source prop (although we could also do this from the TNode src attribute), and mangle it to get our "thumbnail" URI.

By Reusing Internal Building Blocks#

You can reuse to your advantage some building blocks of the internal renderer thanks to the following exports:

  1. IMGElementContainer: to render the container of the <img> element.

  2. useIMGElementProps: To transform generic custom renderer props in props used for the internal image component.

  3. useIMGElementState: To get the state of the image resource fetching.

  4. IMGElementContentError: To render the fallback view on error state.

  5. IMGElementContentLoading: To render the fallback view on loading state.

  6. IMGElementContentSuccess: To render the image on success state.

In the below example, our custom renderer will display an activity indicator when the state is either "loading" or "success".
